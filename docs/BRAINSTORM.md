# 趣味提案アプリ 徹底ブレインストーミング

手法: superpowers流(問題空間の深掘り → 発散 → 前提への挑戦 → 収束と意思決定ポイント)

---

## 1. 問題空間の深掘り — 「なぜ人は趣味を見つけられないのか」

根本原因を5つに分解:

1. **選択肢過多のパラドックス**: 趣味候補は無数にあるが、比較軸がないので選べない。
   「何が好きか」より先に「自分がどういう人間か」の言語化ができていない。
2. **初期投資の不確実性**: 「始めるのにいくらかかるか」「失敗したら道具が無駄になる」
   という金銭的リスクが心理的障壁になる。
3. **孤独の壁**: 一人で始めても続かない。仲間・コミュニティの探し方を知らない。
   特に引っ越し直後・転職直後・退職後に顕著。
4. **継続の壁**: 始めても3週間で辞める。上達実感がない、生活リズムに組み込めない。
5. **「趣味=立派なもの」という思い込み**: 完璧主義が着手を妨げる。
   「散歩」や「銭湯めぐり」も趣味だと認めるハードル設計が必要。

→ **洞察**: 提案(マッチング)だけ解いても価値は薄い。
「不安の除去(お金・孤独)」と「最初の一歩の具体化」までがプロダクトの本体。
現行計画が初期投資+コミュニティ+3ステップを含むのは正しい方向。

## 2. ペルソナ発散

| ペルソナ | 状況 | 刺さる機能 |
|---|---|---|
| 平日消耗型社会人(28) | 週末に何もせず後悔 | 低予算・低時間で始められる提案、週2hフィルタ |
| 転勤・引っ越し直後(35) | 地元に知り合いゼロ | 地域コミュニティ検索、イベントカレンダー |
| 子育て一段落(45) | 自分の時間が突然戻ってきた | 昔好きだったことからの逆算診断 |
| 定年前後(62) | 時間はあるがきっかけがない | 教室・サークル重視、健康軸の提案 |
| 学生(20) | お金がない、時間はある | 初期投資5千円以下フィルタ、無料趣味特集 |
| カップル・夫婦 | 共通の趣味が欲しい | 2人分の診断の重ね合わせ提案 |

## 3. 診断方式の発散(どうやって特性を知るか)

- **A. 質問式アンケート**(現行計画): 10問5軸。実装が軽い、透明性が高い。
- **B. スワイプ式**: 趣味の写真をTinder風に好き/嫌いスワイプ→嗜好を暗黙学習。
  楽しいUXだが、既知の趣味に引っ張られ「意外な発見」が弱い。
- **C. AIチャット対話式**: LLMと会話しながら深掘り。柔軟だがAPI費用と静的ホスティング不可。
- **D. 性格診断ベース(Big Five風)**: 性格→趣味の対応表。学術的納得感があるが質問数が多い。
- **E. 過去体験の逆算**: 「子供の頃、時間を忘れて夢中になったことは?」系の内省質問。
  Cとの組み合わせで強力。

→ ハイブリッド案: **A を骨格に E の内省質問を2〜3問混ぜる**のが費用対効果最大。
B はスワイプで結果を微調整する「第2ラウンド」として将来追加可能。

## 4. 推薦ロジックの発散

- ルールベース5軸マッチ(現行): 説明可能・テスト可能・オフライン動作。
- 「意外性スロット」: 上位6件中1件は「あえて軸を1つ反転させた挑戦枠」を混ぜる
  (セレンディピティ設計。フィルターバブル防止)。
- 「組み合わせ提案」: ソロ趣味×グループ趣味を1つずつ勧める「趣味ポートフォリオ」概念。
- 季節連動: 診断月に応じて季節の趣味(春=登山入門、冬=味噌づくり)をブースト。
- 協調フィルタリング/LLM: ユーザーデータかAPIが必要。静的SPAの範囲外 → v2以降。

## 5. 「お金の不安」を解く機能の発散

- 初期投資レンジ+道具リスト(現行計画)
- **スターターキット3段階**: 「まず無料で試す」「5千円お試しセット」「本格スタート」
  の3プランを趣味ごとに提示 ← 心理的障壁を最も下げる
- 中古・レンタルの選択肢明記(メルカリ/ジモティー/モンベルレンタル等の検索リンク)
- 月額シミュレータ: 「この趣味を1年続けたら総額いくら」の試算表示
- 「無料体験から始める」バッジ: 自治体の体験教室・見学可サークルの存在を示す

## 6. 「孤独の壁」を解く機能の発散

- 地域×趣味の検索リンク生成(現行計画: ジモティー/こくちーず/Maps/X)
- 「一人でも成立する度」スコアの表示(ソロ耐性が低い趣味は仲間必須と明示)
- オンラインコミュニティ(Discord/Reddit/YouTube)の探し方も併記 → 地方在住者の救済
- v2: 都道府県別の定番スポットDB(例: 東京×ボルダリング=主要ジム名)

## 7. 「継続の壁」を解く機能の発散(v2候補)

- 3週間チャレンジ: 始めた趣味の21日間チェックイン(localStorageで完結可能)
- 「やめてもいい」設計: 合わなかったら再診断を促すポジティブな導線。
  「挫折」ではなく「データが増えた」と再フレーミング
- 趣味ログ: 写真1枚+一言の記録。振り返りが継続動機になる

## 8. 前提への挑戦(あえて逆を考える)

- 「診断は正確であるべき」→ 逆: 診断は**会話のきっかけ**でよい。占い的な楽しさ
  (シェアしたくなる結果画面、「あなたは○○タイプ」の命名)が拡散を生む。
- 「多くの趣味を網羅すべき」→ 逆: 40件を**深く**(始め方・費用・コミュニティが具体的)
  の方が、200件を浅くより価値が高い。
- 「アプリが答えを出す」→ 逆: 最終決定は人間。上位6件から選ばせる行為自体が
  納得感(IKEA効果)を生む。

## 9. リスクと対策

| リスク | 対策 |
|---|---|
| 診断結果が凡庸(誰でもウォーキング) | 意外性スロット+カテゴリ分散の強制 |
| 外部リンク切れ(検索URL仕様変更) | 検索URLは単純なクエリ形式のみ使用、テストで形式検証 |
| 費用データの陳腐化 | レンジ表示+「目安」明記、データを1ファイル集約で更新容易に |
| 地方でコミュニティが見つからない | オンラインコミュニティ併記で必ず受け皿を用意 |

## 10. 収束: ロードマップ案

- **MVP(現行計画+ブレスト反映)**: 質問式診断(内省質問入り)、5軸推薦+意外性スロット、
  趣味DB40件(スターターキット3段階つき)、地域+オンラインコミュニティリンク、
  タイプ名つき結果画面、localStorage保存
- **v2**: 3週間チャレンジ、スワイプ微調整、都道府県別スポットDB、2人診断(共通趣味)
- **v3**: Claude API による対話式深掘り診断・パーソナライズ提案文(要ホスティング再考)
